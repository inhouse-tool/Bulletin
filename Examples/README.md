# Examples

## 動作確認用ファイル例

### [Nozomi015.txt](Nozomi015.txt)

このテキストファイルを Daily Schedule として加えると,
朝方, 東京を出発して, 昼に博多に到着するというケースの車内案内が見られます.

### [Nozomi054.txt](Nozomi054.txt)

このテキストファイルを Daily Schedule として加えると,
夕方, 博多を出発して, 夜に東京に到着するというケースの車内案内が見られます.

### [Nozomi.xlsx](Nozomi.xlsx)

上記二つのケースを捻り出した元ネタのエクセルファイルです.

使い方は:

1. A1 のセルにあるコンボボックスで便を選ぶ.
1. 3行目以下の車内案内のテキストを, A列・B列まとめて選択状態にして copy する.
1. メモ帳などに paste する.
1. 適当な名前の .txt ファイルとして save する.
1. 本アプリに Daily Schedule として喰わせる.

という手順です.

せっかく新幹線の電光掲示板風の表示にしたのですから,
やはりこういうことはきちんとやっておきませんと,
日本全国に約1億8000万人は居るという ( 主催者発表 ) 乗り鉄の皆様に叱られてしまいます.

このエクセルの中身は, 2024年秋現在の時刻表を見ながら手打ちしたものなので,
どこかに打ち間違いがあったらごめんなさい.
また, ダイヤが改定されるとズレることになります.
まあ, どうせ気分を味わうためだけのものですから, 大目に見ていただければと.

アナウンスの文言については,
[DenDen-lab さんのページ](https://denden-lab.com/tokaido-shinkansen-nozomi-announce/)
を参考にさせていただきました.

駅通過時刻は停車駅間の所要時間と駅間の実キロ数から割り出した ( 粗っぽい ) 比率上の数値です.
このキロ数は, [こちらのページ](https://tanukiacademy.com/Shinkansen/)
に上げられていたものを参考にさせていただきました.

なお, 本アプリから時刻表のサイトに直接アクセスして情報を得る方式を採らなかったのは,

* Web を読む場合は, サイトのデザインが変更されると動作しなくなる. ( 著作権的にもアレだし. )
* 変更の懸念のない API の場合は, サービス自体が課金されている. ( 駅通過時刻までは未サポートとか. )
* ユーザーが指定した以外のサイトに勝手にアクセスするのは, セキュリティ的に好ましくない.
* 好みの日課をただのテキストファイルとして食わせる方が, 本アプリの汎用性は保てる.

などの理由によるものです.
